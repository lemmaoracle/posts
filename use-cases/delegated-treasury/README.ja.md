# ユースケース: 委任型トレジャリー

## テーゼ

**承認されている ≠ 証明されている**

企業がAIエージェントに予算と業務範囲を渡します。月 10,000 ドルまで、SaaS サブスクのみ、500 ドル超は要承認──この制約はソフトプロンプトに置けません。Lemmaは、発行組織の署名付きで、これらの制約をオンチェーンの支出制御アテステーションとして発行します。決済の相手側はそのアテステーションを検証してから支払いを受け入れます。

## Lemmaが証明するもの

- 発行主体（組織・部門）
- 支出上限とスコープ
- 有効期間
- 失効ステータス

## 問題

企業がAIエージェントに支出権限を委譲するとき、制約は典型的に次のいずれかで実装されます:

1. **ソフトプロンプト** — 「500 ドル以上使わないで」とエージェントに指示します。これは提案であって保証ではありません。プロンプトインジェクション、コンテキストオーバーフロー、または敵対的交渉によって無効化されます。

2. **プラットフォームガードレール** — エージェントプラットフォームが制限を強制します。しかし、これらは中央集権的で不透明、かつポータブルではありません。売り手は、プラットフォームを信頼することなく、買い手の権限を独立して検証する手段を持ちません。

どちらのアプローチも、エージェントの支出権限が本物で、現在有効で、制約付きであることの暗号論理的証明を提供しません。その証明がなければ、売り手はエージェントの主張を信じるか、取引を拒否するしかありません──どちらも最適ではありません。

## Lemmaがどう変えるか

Lemmaは委譲エージェントに**検証可能な支出権限**を提供します:

1. **委譲時:** 発行組織がオンチェーンの支出制御アテステーションを作成します:主体の本人性、支出上限、カテゴリスコープ、有効期間、失効エンドポイント。
2. **取引時:** 売り手エージェントは支払いを受け入れる前にアテステーションを検証します。アテステーションは発行組織によって署名されています──エージェントでもプラットフォームでもありません。
3. **監査時:** すべての取引が、それを認可した委譲の暗号論理的証明を伴います。コンプライアンスチームは検証可能な証拠を得ます。ログエントリではありません。

委譲が証明であり、アテステーションが契約であり、オンチェーンのアンカーが保証です。
