# アーキテクチャ: オンチェーン支出制御アテステーション

## 設計原則

**委譲が証明であり、アテステーションが契約であり、オンチェーンのアンカーが保証である。**

```
┌─────────────────────────────────────────────────────────────┐
│                     企業（主体）                               │
│                                                              │
│  ┌──────────────────┐        ┌──────────────────────────┐   │
│  │  委譲ポリシー      │───────▶│  Lemma アテステーション発行  │   │
│  │  - 月$10K上限      │        │  - 権限のZK証明            │   │
│  │  - SaaSのみ       │        │  - オンチェーンアンカー      │   │
│  │  - $500/取引上限   │        │  - 失効エンドポイント       │   │
│  └──────────────────┘        └────────────┬─────────────┘   │
└────────────────────────────────────────────┼─────────────────┘
                                             │ アテステーション
                                             ▼
┌─────────────────────────────────────────────────────────────┐
│                     エージェント（受任者）                      │
│                                                              │
│  ┌──────────────────┐        ┌──────────────────────────┐   │
│  │  購入リクエスト    │───────▶│  アテステーションキャリア    │   │
│  │  - $299 SaaS sub  │        │  - 取引に証明を添付         │   │
│  │  - スコープ内      │        │  - 有効性が検査可能          │   │
│  └──────────────────┘        └────────────┬─────────────┘   │
└────────────────────────────────────────────┼─────────────────┘
                                             │ 支払い + 証明
                                             ▼
┌─────────────────────────────────────────────────────────────┐
│                     売り手（相手方）                           │
│                                                              │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  Lemma 検証                                           │   │
│  │  1. アテステーションは主張された発行者によって署名されているか？ │   │
│  │  2. 支出は記載された上限内か？                           │   │
│  │  3. カテゴリは記載されたスコープ内か？                    │   │
│  │  4. アテステーションはまだ有効か（失効していないか）？      │   │
│  │  ✓ すべてのチェックが通過 → 支払いを受け入れる             │   │
│  │  ✗ いずれかのチェックが失敗 → 取引を拒否                   │   │
│  └──────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

## コンポーネントフロー

1. **発行:** 企業の委譲ポリシーがLemmaアテステーションとしてエンコードされる。Poseidonハッシュが制約をオンチェーンにコミット；BBS+署名が発行者の権限を証明する。
2. **運搬:** エージェントは各取引にアテステーションを携える。証明はポリシーの全体を開示しない——特定の取引が認可された範囲内にあることのみを示す。
3. **検証:** 売り手はアテステーションを独立して検証する。エージェントやプラットフォームへの信頼は不要——発行組織の署名のみを信頼する。
4. **失効:** 企業が委譲を取り消すと、オンチェーンの失効ステータスが即座に更新される。売り手はリアルタイムで失効を確認する。
