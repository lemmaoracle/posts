# ユースケース: 経費承認を、AI エージェントに任せる。

## テーゼ

**委任されている ≠ 範囲が証明されている。**

経費承認を AI エージェントに任せたい。だが「上限 ¥100k まで」「出張と備品だけ」「課のメンバーの分だけ」といった制約は、プロンプトに書いても保証にはなりません。範囲は、検証可能な委任として外側に固定される必要があります。

## Lemma が証明するもの

- 委任元（組織・部門）と委任先エージェント
- 承認できる上限額（spendLimitUSDC）
- カテゴリ・担当範囲（scope）
- 有効期限と失効ステータス

## 問題

経費承認を自動化するとき、権限の制約は典型的に次のいずれかで実装されます。

1. **ソフトプロンプト** — 「¥100k を超えたら承認しない」と指示するだけ。プロンプトインジェクションや文脈あふれで簡単に外れ、超過承認が通ってしまいます。
2. **承認システム内のルール** — 中央のワークフローに上限を埋め込む。だが監査側からは「その承認が、どの権限委任に基づいていたか」を後から独立に検証できません。

どちらも、AI が下した承認が「正規の委任の範囲内だった」ことの暗号論理的な証明を残しません。証明がなければ、内部統制も監査も、ログを信じるしかありません。

## Lemma がどう変えるか

Lemma は経費承認エージェントに**範囲付きの検証可能な権限**を与えます。

1. **委任時:** 経理・財務部門が、上限額・カテゴリ・担当範囲・有効期限を持つ委任証明を発行します。署名するのは発行組織であり、エージェントでもプラットフォームでもありません。
2. **承認時:** エージェントは承認を出す前に委任の範囲を runtime で検証します。範囲内なら自律承認、範囲外は人間にエスカレーションして止まります。
3. **監査時:** すべての承認に、それを認可した委任の証明が付きます。承認権限の中身を開示せずに、「範囲内だった」ことを当局・監査人に示せます。

委任が証明であり、範囲が契約です。範囲外は、実行前に止まります。
