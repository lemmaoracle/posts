# ユースケース: エージェント間の取引を、権限チェーンで証明する。

## テーゼ

**取引が成立した ≠ 委任が連鎖して正当だった。**

自律エージェント同士が取引する A2A 環境では、ひとつの取引の裏に複数の委任が連なります。最終的な支払いが「どの組織から、どのエージェントへ、どこまで委任された権限に基づくのか」を、連鎖として辿れる必要があります。

## Lemma が証明するもの

- 委任元（root 組織）から各エージェントへの委任の連鎖（chain）
- 各エージェントの役割（role）と範囲（scope）
- 有効期限と失効ステータス
- 最終取引が連鎖の範囲内であること

## 問題

A2A 取引では、権限の連鎖は典型的に次のいずれかで扱われます。

1. **委任関係が暗黙** — どのエージェントが誰の権限で動いているかが、ログを突き合わせないと分かりません。取引の正当性を第三者が独立に検証できません。
2. **プラットフォーム内に閉じる** — 単一基盤の中でだけ委任が管理される。基盤をまたぐ取引や、外部監査には持ち出せません。

どちらも、最終取引が「正当な委任の連鎖に基づいていた」ことの証明を残しません。

## Lemma がどう変えるか

Lemma は A2A 取引に**辿れる委任の連鎖**を与えます。

1. **委任時:** root 組織 → エージェント → サブエージェントの階層的な委任を、それぞれ署名付きの証明として発行します。
2. **取引時:** 取引の前段で、委任の連鎖（chain）が範囲内かを runtime で検証します。連鎖が途切れていれば、取引は成立しません。
3. **監査時:** 最終取引から委任元までを chain として辿れます。委任関係や取引内容の中身を開示せずに、「誰が誰に委任したか」と取引の正当性を示せます。

委任の連鎖が証明であり、各ステップの範囲が契約です。連鎖が辿れない取引は、成立しません。
