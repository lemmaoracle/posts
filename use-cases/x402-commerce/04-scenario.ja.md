---
title: "実例：在庫詐称する一時的売り手エージェント"
---


エージェント・コマースのマーケットプレイスで、買い手エージェントが特定の半導体在庫を探しています。x402 プロトコル経由で複数の売り手エージェントから即座にオファーが届きます。あるオファーは在庫 1,000 個・最低価格・48 時間納品 SLA を主張。買い手は決済し、x402 はミリ秒で支払いを完了させます。

48 時間後、納品はゼロ。問い合わせると、その売り手エージェントは既に廃止されており、責任主体が見当たりません。マーケットプレイスのレピュテーションシステムは、その売り手にネガティブスコアを付与しますが、買い手の損失はすでに発生済み。一時的なエージェントが乱立する世界では、このパターンは構造的に避けられません。

Lemma が導入されていれば、売り手エージェントが提示する在庫・価格・SLA・本人性の各属性が ZK 証明で包まれており、買い手エージェントは決済コミット前にこれらを検証します。在庫証明の発行者が信頼できないか、SLA 約定の権限証明が揃っていなければ、x402 決済は走らず、境界で停止します。

業種別のユースケース設計、x402 / Stripe Agent SDK / MCP との統合パターン、エージェント身元証明の発行スキーマは、相談コール後にお渡しする業種別キットでご共有しています。
