Lemma は x402 プロトコルや既存のエージェント・コマース実装を置き換えません。売り手の属性提示と買い手の決済コミットの間に、ZK 証明検証のゲートを 1 段挟むだけです。

売り手側は出品時に各属性を ZK 証明として発行し、買い手側は受信時にこれを検証します。属性ごとに発行者・有効期限・スコープが暗号的に固定されるため、なりすまし・在庫詐称・SLA 偽装は構造的に検知され、x402 決済まで到達しません。事後の取引証跡は両者に残り、紛争解決の暗号的根拠になります。

x402 / Stripe Agent SDK / MCP との統合パターン、属性証明発行スキーマ、マーケットプレイス運営者向けの組み込みパターンは、ホワイトペーパーおよび相談コール後の技術資料で詳述しています。
