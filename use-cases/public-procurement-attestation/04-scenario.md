---
title: "Scenario: a municipal IT-system procurement"
---


A mid-size municipality runs a tender for a citizen-facing IT system. The tender's eligibility schema requires ISO 27001 certification, a financial-adequacy threshold, three or more references with public-sector scope in the last five years, and a personal-data-protection self-assessment matching the municipality's reference framework.

In the document-based workflow, every bidder submits a PDF pack against each requirement. The procurement team takes custody of certifications, balance sheets, reference letters from other public agencies, and personnel data attached to reference projects. A single bidder pack can run to several hundred pages. The evaluator reads only the parts that map to the eligibility predicate; the rest sits in the municipality's filing system as a permanent disclosure liability.

With per-attribute attestation, each bidder presents a single composed ZK proof against the eligibility schema. The evaluator verifies the predicate and records the verification in the tender's audit trail. The underlying ISO certificate stays with the certifying body, the financial records stay with the auditor, and the reference letters stay with the reference organizations. The audit trail is independently verifiable by an oversight body without re-reading the bidder packs.

Tender-category eligibility schemas, issuer-side onboarding for certifying bodies and auditors, the evaluator-side workflow integration, and the audit-trail interface for oversight bodies live in the post-Discovery materials.
