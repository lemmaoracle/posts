# シナリオ: 出向従業員によるデータ流出

## Lemma導入前 — インシデントが発生するまで

### 背景

生命保険会社（保険会社A）が従業員を銀行代理店（代理店B）に出向させ、銀行のチャネルを通じて保険商品を販売します。この従業員は:

1. 保険会社Aの顧客データベースへの認証情報を持っている
2. 代理店Bの顧客管理システムへの認証情報を取得する
3. 代理店Bのオフィスで両方のシステムに同時にアクセスしながら働く

### インシデントのタイムライン

| 段階 | 発生すること | 検知できない理由 |
|---|---|---|
| アクセス | 従業員が両システムにまたがる2,476件の顧客レコードを照会 | アクセスは正当に見える — 従業員は依然として有効な認証情報を持つ |
| 収集 | 従業員がデータを個人端末/USB/クラウドストレージにコピー | 改ざん検知可能なロギングがない — アクセスログは変更可能で、分断されたシステムに保存される |
| 持ち出し | 従業員がデータを新たな雇用先に持ち込む、または売却する | 組織間の監査証跡が存在しない |
| 発覚 | 数ヶ月後、顧客の苦情や規制当局の監査により侵害が発覚 | ログは改ざんされている可能性がある; 元のアクセスパターンの暗号学的証明がない |
| 対応 | 保険会社Aと代理店Bは、何がいつアクセスされたかを正確に証明できない | 相互の非難; 両組織への規制上の罰則 |

### 根本原因分析

1. **信頼境界のあいまいさ** — 組織間境界における明確で強制可能なデータアクセスポリシーがない
2. **変更可能な監査ログ** — 両組織がアクセスログを変更可能なデータベースに保存; 完全性の暗号学的保証がない
3. **共有証明レイヤーの欠如** — 保険会社Aは代理店Bのログを検証できず、逆も同様
4. **抑止力のギャップ** — 従業員はログが争えることを知っている; 「否認できない証拠」という抑止力がない

---

## Lemma導入後 — 同じシナリオがどのように変わるか

### 導入時の変化

1. **Lemma認証ゲートウェイ**が各組織のCRM/データベースとユーザーの間に配置される
2. すべてのデータアクセス要求が **ZK認証**を生成: 「ユーザーXがタイムスタンプTにレコードYにアクセスした」 — レコード内容を明かさず
3. 認証はオンチェーン（またはコミットメントルート）に固定され、**構成上改ざん検知可能**になる
4. 両組織は**検証可能な証明レイヤー**を共有 — どちらも一方的にアクセス履歴を変更できない

### インシデントタイムライン（Lemma導入時）

| 段階 | 発生すること | Lemmaによる変化 |
|---|---|---|
| アクセス | 従業員が顧客レコードを照会 | 各照会がZK認証を生成し、タイムスタンプ付きでコミットされる |
| 収集試行 | 従業員がデータの一括エクスポートを試みる | 異常なアクセスパターン（量、タイミング）がDLPによってフラグ付け; Lemma認証はどのレコードにアクセスされたかを正確に証明 |
| 抑止 | 従業員はすべてのアクセスが暗号学的に証明されることを認識 | 心理的抑止力 — 「この証跡を否定したり変更したりできない」という認識が内部脅威の動機を減らす |
| 流出が発生した場合 | 従業員がデータを持ち去る | 両組織は何がアクセスされたかの**改ざん不可能な共有証明**を持つ — 紛争の余地なし |
| 規制対応 | 金融庁が監査証拠を要求 | 保険会社が検証可能な認証を提供 — 手動でのログ調整は不要 |

### 定量化された影響

| 指標 | Lemmaなし | Lemmaあり |
|---|---|---|
| 異常アクセスの検知時間 | 数週間〜数ヶ月 | リアルタイム (DLP) + 暗号学的証明 (Lemma) |
| アクセスログの争議可能性 | 高い — ログは変更可能で分断されている | ほぼゼロ — ZK認証は改ざん検知可能 |
| 規制監査の準備 | 数週間の手動ログ集約 | 数時間 — 検証可能な認証セットをエクスポート |
| 組織間の責任のなすりつけ | 相互の否定 | 共有された否認できない証明 |
| 内部脅威の抑止力 | 低い — 「私はおそらく否定できる」 | 高い — 「すべてのアクセスが暗号学的に証明される」 |

## 営業ナラティブのための主要シナリオ

### シナリオA: 「規制当局が質問してきた」

> 金融庁が報告命令を発出。Lemmaなし: 36の代理店にまたがる2週間のログ・フォレンジクス。Lemmaあり: 1時間以内に認証セットをエクスポートし、暗号学的に検証可能。

### シナリオB: 「出向従業員」

> 従業員が保険会社Aから代理店Bに出向。Lemmaは時間制限付きスコープを持つ属性ベースのアクセス認証情報を発行します。出向が終了すると、アクセスは失効し、出向中のすべてのアクセスは証明可能にログ記録されます。

### シナリオC: 「組織間監査」

> 2つの組織が誰が何にアクセスしたかを調整する必要がある。Lemmaなし: 各組織が独自のログをエクスポートし、相手のログを検証する方法がない。Lemmaあり: 共有コミットメントルート、相互検証可能性。
