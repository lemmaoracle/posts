# 問題提起: 証明のない信頼境界

出向社員が組織間を移動するとき、顧客データに対する信頼境界は曖昧になる。出元も受け入れ先も有効な認証情報を付与するが、どちらも従業員が何にアクセスしたかを暗号論理的に証明できない——アクセスログが改ざんされていないことも。

MetLife日本（2026）の事案はこのギャップを露呈した：出向社員が36の代理店パートナーから2,476件の顧客記録を持ち出した。DLPは異常を検出した。SIEMはイベントを記録した。しかしFSAが証拠を求めたとき、両組織が提供できたのは、変更可能でサイロ化されたログだけだった——異議申し立てが可能で、改ざん可能で、不完全なログ。

**検出はある。証明はない。** それがLemmaが埋める構造的ギャップである。
