# ユースケース: AI モデル切替時の動作整合性を、後から証明する。

## テーゼ

**モデルは更新される ≠ 過去の判断が再現できなくなる。**

AI モデルを更新すると、「その時のモデルで、その時のポリシーで、何を判断したか」が曖昧になります。過去の判断を遡って再現・検証できなければ、AI ガバナンスの責任は宙に浮きます。判断時点を、固定できる必要があります。

## Lemma が証明するもの

- 判断時点のモデル（modelId@timestamp）
- 判断時点のポリシー（policyHash）
- 入力・出力のコミットメントと適合
- 判断が記録された時刻（recordedAt）

## 問題

モデル更新時の整合性は、典型的に次のいずれかで扱われます。

1. **バージョン番号のログ** — どのモデルを使ったかを記録する。だが「その版・そのポリシーで、その入力に対し、その判断を下した」ことを暗号的に固定できません。
2. **モデルの保存** — 旧モデルをアーカイブする。だが運用環境・ポリシー・前後関係まで再現するのは難しく、第三者が独立に検証できません。

どちらも、過去の判断を「時点固定で、遡及的に、検証可能」に再現する構造を持ちません。

## Lemma がどう変えるか

Lemma は AI の判断ごとに**時点固定のコミットメント**を発行します。

1. **版固定:** modelId@timestamp と policyHash を、判断の瞬間にコミットします。
2. **判断時:** 入力・出力のコミットメントと satisfiesPolicy を同時に固定します。モデルが更新されても、この記録は不変です。
3. **遡及検証:** モデル更新後も、過去の判断を「その時のモデル・その時のポリシー」で遡及検証できます。モデル内部やパラメータを開示せずに、判断の整合性を独立に示せます。

判断時点が証明であり、modelId@timestamp が固定点です。モデルは変わっても、過去の判断は遡れます。
