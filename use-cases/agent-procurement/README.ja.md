# ユースケース: 発注・購買を、AI エージェントに任せる。

## テーゼ

**発注権限は、丸投げではなく範囲で渡す。**

定型発注を AI エージェントに任せたい。だがベンダ選定権をまるごと AI に預けることはできません。「認証済みベンダだけ」「カテゴリは備品」「1 件 ¥500k まで」という範囲を、検証可能な委任として固定する必要があります。

## Lemma が証明するもの

- 委任元（調達・購買部門）と発注エージェント
- 発注できる上限額（spendLimitUSDC）
- 認証済みベンダ・カテゴリの範囲（scope）
- 有効期限と失効ステータス

## 問題

発注を自動化するとき、権限の制約は典型的に次のいずれかで実装されます。

1. **ソフトプロンプト** — 「認証済みベンダにだけ発注して」と指示するだけ。敵対的な入力や文脈あふれで外れ、想定外のベンダ・金額が通ってしまいます。
2. **購買システム内のルール** — 基幹システムに上限を埋め込む。だが監査側からは「その発注が、どの権限委任に基づいていたか」を後から独立に検証できません。

どちらも、AI が確定した発注が「正規の委任の範囲内だった」ことの証明を残しません。

## Lemma がどう変えるか

Lemma は発注エージェントに**範囲付きの検証可能な権限**を与えます。

1. **委任時:** 調達・購買部門が、上限額・認証済みベンダ・カテゴリ・有効期限を持つ委任証明を発行します。署名するのは発行組織です。
2. **発注時:** エージェントは発注を確定する前に委任の範囲を runtime で検証します。範囲内なら自律発注、範囲外は止まります。
3. **監査時:** すべての発注に、それを認可した委任の証明が付きます。取引先データや予算の中身を開示せずに、「範囲内だった」ことを示せます。

委任が証明であり、範囲が契約です。範囲外の発注は、確定前に止まります。
