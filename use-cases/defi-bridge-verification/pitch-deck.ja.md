# ピッチデッキ: DeFiブリッジ検証 — ワンスライド

---

## 46分で$292Mが盗難。出所検証はゼロ。

**問題:** Kelp DAOのブリッジは、単一のDVN署名に基づいて受信システムがコミットしたために流出した。署名は有効だった。メッセージは偽物だった。「暗号論理的に有効 ≠ 意味的に正しい」

### 構造的ギャップ

```
今日のブリッジ:
  DVN署名 ──────────────────▶  コミット
                  ↑ 信頼の単一障害点

Lemmaあり:
  DVN署名  ──┐
  出所証明   ──┼──────────────▶  コミット
                  ↑ 2つの独立した検証
```

### Lemma導入前

| ステップ | 何が起きるか | ギャップ |
|---|---|---|
| 攻撃者がクロスチェーンメッセージを偽装 | 暗号論理的に有効、意味的に誤り | 出所検証なし |
| 1-of-1 DVNが承認 | 単一署名が信頼される | 独立したチェックなし |
| OFTアダプタが$292Mをリリース | DVNのみに基づいてコミット | 事前実行アテステーションなし |
| 46分後: 緊急一時停止 | 既に被害は発生 | 事後対応のみ |

### Lemma導入後

| ステップ | 何が起きるか | メリット |
|---|---|---|
| 攻撃者がクロスチェーンメッセージを偽装 | 暗号論理的に有効、意味的に誤り | 同じ |
| 1-of-1 DVNが承認 | 単一署名が信頼される | 同じ |
| **Lemma出所証明: 拒否** | **偽メッセージが独立検証に失敗** | **境界でエクスプロイト停止** |
| **$292Mはエスクローに留まる** | **コミットなし、波及なし、不良債権なし** | **ゼロ分** |

### レイヤー

```
┌──────────────────────────────────────────────┐
│  事後対応    │  フォレンジック / 凍結 / リカバリー  │  ← 症状を捉える
├──────────────┼─────────────────────────────────┤
│  事前実行    │  Lemma出所検証                    │  ← コミットを防ぐ
└──────────────┴─────────────────────────────────┘
```

**事後ツールは症状を捉える。Lemmaはコミットを防ぐ。**
