### KYC/AML Selective Disclosure — the broader financial pattern
**Declared ≠ verified**

The general selective-disclosure pattern for financial services. LP attestation is one application; the same shape applies to KYC at onboarding, cross-border attribute checks, and AML-tier disclosure.

[See the use case →](/use-cases/kyc-aml-selective-disclosure/)

### DeFi Bridge Verification — verifiable cross-chain state

For protocols that bridge between chains, the bridge-side state needs the same attestation discipline as the LP-side identity layer. Different attributes, same trust pattern.

[See the use case →](/use-cases/defi-bridge-verification/)

### Financial Data Exfiltration — verifiable access logs
**Access logs exist ≠ untampered**

When a regulator audits the protocol, the access logs themselves need to be provably untampered. The same attestation primitive applies to operational records, not just LP records.

[See the use case →](/use-cases/financial-data-exfiltration/)
